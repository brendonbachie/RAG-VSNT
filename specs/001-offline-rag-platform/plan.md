# Implementation Plan: Offline RAG Commercial Platform

**Branch**: `001-offline-rag-platform` | **Date**: 2026-05-16 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/001-offline-rag-platform/spec.md`

## Summary

Transform the existing terminal-based RAG VSNT system into a multi-user, browser-accessible
commercial product for organizations querying classified offline documentation. The core
retrieval pipeline (BGE embeddings + BM25 hybrid search + cross-encoder reranking via
LlamaIndex) is preserved unchanged. A FastAPI web application layer adds user authentication,
role-based access control, a web chat interface, document upload, audit logging, and runtime
model switching — all 100% offline on a single machine.

## Technical Context

**Language/Version**: Python 3.11+

**Primary Dependencies**:
- `fastapi` + `uvicorn[standard]` — async web server, routing, file upload
- `jinja2` + `python-multipart` — server-side HTML templates, form/file handling
- `sqlalchemy` + `aiosqlite` — async ORM over SQLite
- `passlib[bcrypt]` — password hashing
- `itsdangerous` — signed session cookies (httponly, no JWT complexity)
- `pdfplumber` — PDF text extraction (pure Python, no system dependencies)
- `python-docx` — DOCX text extraction
- `watchfiles` — async folder watcher for docs/ auto-indexing trigger
- All existing deps: `llama-index-core`, `llama-index-llms-ollama`,
  `llama-index-embeddings-huggingface`, `llama-index-retrievers-bm25`,
  `sentence-transformers`, `torch`, `rank-bm25`, `pyyaml`

**Storage**:
- SQLite (`data/vsnt_rag.db`) — users, audit events, system settings
- `.rag_index/` — existing LlamaIndex vector store (unchanged)
- `docs/` — source documents (unchanged)

**Testing**: pytest + httpx (async test client for FastAPI)

**Target Platform**: Windows 10/11 (Python 3.11+ from python.org), Ubuntu 20.04+,
Debian 11+. Ollama installed separately per existing README.

**Project Type**: localhost web-service (single machine, browser accessed at
http://127.0.0.1:8000)

**Performance Goals**:
- Query response: ≤ 60 s end-to-end (LLM inference dominates; same as current terminal
  baseline)
- PDF indexing: ≤ 5 min for a 50 MB file on recommended hardware
- Web UI page load: ≤ 2 s for all non-query pages

**Constraints**:
- 100% offline at runtime; server bound to 127.0.0.1 only
- No Docker dependency; installation via pip + a single shell/batch script
- Single machine deployment; no distributed architecture
- Must coexist with existing `rag_vsnt_offline.py` (terminal mode)

**Scale/Scope**: 1–50 concurrent users, hundreds of documents, up to a few GB indexed data

**Structure Decision**: Single-project layout. The existing `rag_vsnt_offline.py` terminal
entry point is preserved alongside the new `app.py` web entry point — they share the same
`docs/` and `.rag_index/` directories. The `src/` package contains all new web-layer code.
No `backend/`+`frontend/` split is needed since the UI is server-side rendered with
Jinja2 templates (no frontend build step, keeping installation simple per Principle V).

## Complexity Tracking

> No Constitution Check violations — this section is informational only.

No violations requiring justification. The choice to use FastAPI over Streamlit (already
in requirements) is a simplification: Streamlit's session model does not support
multi-user authentication, which is a P2 requirement. Replacing it avoids a category of
hacks that would violate Principle V.
